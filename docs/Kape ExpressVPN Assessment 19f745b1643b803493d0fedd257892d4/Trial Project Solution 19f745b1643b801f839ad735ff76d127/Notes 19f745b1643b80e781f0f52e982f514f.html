⭐️ Using Docker Containers

  • I used the following reference on creating Ubuntu containers using : https://medium.com/@andrewdass/make-your-own-dockerfile-to-run-an-ubuntu-container-015c58beb869
  • For simplicity, I used Visual Studio Code IDE for creating files and checking created Docker images and containers
  • Alternatively, I checked the images and running containers using Docker Desktop!
  • Additional Info for Dolly:
    • Containers must be started before re-executing it, to check the names of the containers via terminal.
    • See docker manual for common commands: https://docs.docker.com/reference/cli/docker/

⭐️ Using ExpressVPN inside Docker container

  • I tried using the following github repo for installing ExpressVPN inside the Docker container: https://github.com/polkaned/dockerfiles.git

⭐️ Trying existing packages for checking speed

  • I wanted to use Ookla’s speedtest cli for getting speed measurements etc. because it works well with VPN but the limitation for current environment above (setting platform to linux/amd64) blocks the speedtest from running even though it is installed properly in the container (reference: https://github.com/robinmanuelthiel/speedtest.git)
  • After that, I tried using LibreSpeed as well, but even though the package being installed is in amd64 (to be consistent with the set platform), it fails since it somehow still detects the host’s device structure(arm64)
  • I ran out of time to debug the above, so in the end I used python’s speedtest-cli to measure speed
