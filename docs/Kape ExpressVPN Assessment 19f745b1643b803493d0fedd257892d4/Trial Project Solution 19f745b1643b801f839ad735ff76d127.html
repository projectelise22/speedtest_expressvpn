Trial Project Solution

Main Requirements

  • Utility should be able to measure the speed without and with a VPN connection
  • Utility should interact with VPN via command line with proper guidelines
  • Utility’s speed method should be able to measure download and upload speed, and ping
  • Utility should be able to take json input file, and output the same with specific format as described
    • Input file must contain 3 server locations (selection as described)
    • Utility must get the connection time, and speed
    • Get the speed 5 times to get the average speed
    • Output file should include information about environment, speed without vpn, and speed with vpn

Utility Features

  1. Utility script is written in python named speedtest_util.py.
  1. It requires an input file named locations.json with 3 server locations.
  1. Contains vpn_aliases.json which is a list of all valid server locations serviced by ExpressVPN in json format.
  1. Running the script returns two files:
    • Output results with filename results_<date_time>.json
    • Output log from the above results with filename speedtest_util_<date_time>.log
    • Easier to check the logs with corresponding output since both will have the same date_time
  1. As mentioned above utility has logging features to check the following errors if they occur:
    • error loading input json file
    • no vpn alias found for the selected country + city
    • failed connection to selected location after maximum retries
    • failed speedtest after maximum retries
  1. Logging also shows the status for each process of server location
  1. Utility Information and Methods
    • Utility is created as a class which has methods for:
      • set up logging
      • running a shell command
      • loading json files
      • getting vpn status
      • getting vpn alias
      • measure connecting time
      • getting the speed (in this case, download speed from the results of the speedtest command)
      • getting the machine/env information
      • testing each vpn location (main method)

Test Strategy

  1. I tested mainly the method of the script to ensure that I get results for each server location, and that errors are handled when they occur. Tests can be found under util/tests/
  1. Testing strategy was fairly simple, I tried to use mock methods to simulate the expected behavior of the functions given some mock return values of when the vpn and speedtest cli commands are run
  1. Testing are done on the following methods:
    • getting vpn status
    • getting vpn alias
    • measure connecting time
    • getting the speed (in this case, download speed from the results of the speedtest command)

Limitations

  • Testing is done only on available device (Mac)
  • Since speedtest-cli used is in python, it is quite slower and sometimes fails in connecting
